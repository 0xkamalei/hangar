我订阅了多个代理机场，有的机场是按月有一定额度的
每个机场的IP稳定性都不一样
我实际为了使用一些服务，总是要手动切换机场和在clash verge中切换代理


为了解决这个问题
我决定自己制作一个sub订阅合并管理工具

将多个订阅地址合并成一个 并且根据我的需求制定group 并且需要合并 basic.yml中的规则和使用 #规则.md 这个规则集

这是一个 基于 tauri2.0的 Mac程序

程序的逻辑是：
1. 用户可以添加多个代理订阅地址
2. 把代理IP 按照地区分可用组 在原来名称的基础上加上 机场名字
3. 针对我经常需要用的服务创建可用组，比如 Google，Gemini，chatgpt, 等
4. 合并basic 中的规则  这是一般的代理规则
5. 最后生成一个 clash verge 可用的 yaml 文件；
6. 启动一个 server 返回这个文件
